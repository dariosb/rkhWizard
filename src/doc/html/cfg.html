Configuration

Prev: Home
Next: Representing a state machine: step by step

Each RKH application must have its own configuration file, called rkhcfg.h. This file adapts and configures RKH by means of compiler definitions and macros allowing to restrict the resources consumed by RKH. Adjusting this definitions allows to reduce the ROM and RAM consumption, and to enhance the system performance in a substantial manner. The rkhcfg.h file shows the general layout of the configuration file. Here is an list of all options with their documentation:


This section includes:

  • Configuration options related to framework
  • Configuration options related to state machine applications
  • Configuration options related to trace facility
  • Configuration options related to queue (by reference) facility
  • Configuration options related to fixed-sized memory block facility
  • Configuration options related to software timer facility

Configuration options related to framework

  • RKH_MAX_SMA

    Specify the maximum number of state machine applications (SMA) to be used by the application (can be a number in the range [1..64]).

  • RKH_EN_DYNAMIC_EVENT

    If the dynamic event support (see RKH_EN_DYNAMIC_EVENT) is set to 1, RKH allows to use event with parameters, defer/recall, allocating and recycling dynamic events, among other features.

  • RKH_MAX_EPOOL

    If the dynamic event support is enabled (see RKH_EN_DYNAMIC_EVENT) then the RKH_MAX_EPOOL can be used to specify the maximum number of fixed-size memory block pools to be used by the application (can be a number in the range [0..256]). Note that a value of 0 will completely suppress the memory pool services.

  • RKH_SIZEOF_EVENT

    Specify the size of the event signal. The valid values [in bits] are 8, 16 or 32. Default is 8. The higher the signal size, the higher the event structure size and therefore more memory consumption. See RKHE_T data type.

  • RKH_SIZEOF_ESIZE

    Specify the data type of event size. The valid values [in bits] are 8, 16 or 32. Default is 8. See RKHES_T, rkh_epool_register(), and RKH_ALLOC_EVENT().

  • RKH_EN_DEFERRED_EVENT

    If the RKH_EN_DEFERRED_EVENT is set to 1 and the dynamic event support is enabled (see RKH_EN_DYNAMIC_EVENT), RKH enables the defer and recall event features.

  • RKH_ASSERT_EN

    If the RKH_ASSERT_EN is set to 0 the checking assertions are disabled. In particular macros RKHASSERT(), RKHREQUIRE(), RKHENSURE(), RKHINVARIANT(), and RKHERROR() do NOT evaluate the test condition passed as the argument to these macros. One notable exception is the macro RKHALLEGE(), that still evaluates the test condition, but does not report assertion failures when the RKH_ASSERT_EN is enabled.

  • RKH_HK_EN_DISPATCH

    If the RKH_HK_EN_DISPATCH is set to 1, RKH will invoke the dispatch hook function rkh_hk_dispatch() when dispatching an event to a SMA. When this is set the application must provide the hook function.

  • RKH_HK_EN_SIGNAL

    If the RKH_HK_EN_SIGNAL is set to 1, RKH will invoke the signal hook function rkh_hk_signal() when the producer of an event directly posts the event to the event queue of the consumer SMA. When this is set the application must provide the hook function.

  • RKH_HK_EN_TIMEOUT

    If the RKH_HK_EN_TIMEOUT is set to 1, RKH will invoke the timeout hook function rkh_hk_timeout() when a timer expires just before the assigned event is directly posted into the state machine application queue. When this is set the application must provide the hook function.

  • RKH_HK_EN_START

    If the RKH_HK_EN_START is set to 1, RKH will invoke the start hook function rkh_hk_start() just before the RKH takes over control of the application. When this is set the application must provide the hook function.

  • RKH_HK_EN_EXIT

    If the RKH_HK_EN_EXIT is set to 1, RKH will invoke the exit hook function just before it returns to the underlying OS/RTOS. Usually, the rkh_hk_exit() is useful when executing clean-up code upon SMA terminate or framework exit. When this is set the application must provide the hook function.

  • RKH_SMA_EN_IEVENT

    If the RKH_SMA_EN_IEVENT is set to 1 then an initial event could be be passed to state machine application when it starts, like an argc/argv. Also, the RKH_SMA_EN_IEVENT changes the initial action prototype.

  • RKH_EN_SMA_THREAD

    If the RKH_EN_SMA_THREAD is set to 1, each SMA (active object) has its own thread of execution.

  • RKH_EN_SMA_THREAD_DATA

    If the RKH_EN_SMA_THREAD and RKH_EN_SMA_THREAD_DATA are set to 1, each SMA (active object) has its own thread of execution and its own object data.

  • RKH_EN_NATIVE_SCHEDULER

    If the RKH_EN_NATIVE_SCHEDULER is set to 1 then RKH will include the simple, cooperative, and nonpreemptive scheduler RKHS. When RKH_EN_NATIVE_SCHEDULER is enabled RKH also will automatically define RKH_EQ_TYPE, RKH_SMA_BLOCK(), RKH_SMA_READY(), RKH_SMA_UNREADY(), and assume the native priority scheme.

  • RKH_EN_NATIVE_EQUEUE

    If the RKH_EN_NATIVE_EQUEUE is set to 1 and the native event queue is enabled (see RKH_RQ_EN) then RKH will include its own implementation of rkh_sma_post_fifo(), rkh_sma_post_lifo(), and rkh_sma_get() functions.

  • RKH_EN_NATIVE_DYN_EVENT

    If the RKH_EN_NATIVE_DYN_EVENT is set to 1 and the native fixed-size memory block facility is enabled (see RKH_MP_EN) then RKH will include its own implementation of dynamic memory management. When RKH_EN_NATIVE_DYN_EVENT is enabled RKH also will automatically define RKH_DYNE_TYPE, RKH_DYNE_INIT(), RKH_DYNE_GET_ESIZE(), RKH_DYNE_GET(), and RKH_DYNE_PUT().

  • RKH_EN_REENTRANT

    If the RKH_EN_REENTRANT is set to 1, the RKH event dispatch allows to be invoked from several threads of executions. Enable this only if the application is based on a multi-thread architecture.

Configuration options related to state machine applications

  • RKH_SMA_EN_ID

    If the RKH_SMA_EN_ID is set to 1 then RKH allows to add a numerical value to every state machine. This number allows to uniquely identify a state machine application.

  • RKH_SMA_EN_GET_INFO

    If the RKH_SMA_EN_GET_INFO is set to 1 then RKH will include the rkh_clear_info() and rkh_get_info() functions.

  • RKH_SMA_EN_STATE_ID

    If the RKH_SMA_EN_STATE_ID is set to 1 then RKH allows to add a numerical value to every state and pseudostate. This number allows to uniquely identify a state and/or pseudostate.

  • RKH_SMA_EN_PPRO

    If the RKH_SMA_EN_PPRO is set to 1 then RKH allows to reference a event preprocessor to any basic state. Aditionally, by means of single inheritance in C it could be used as state's abstract data. Moreover, implementing the single inheritance in C is very simply by literally embedding the base type, RKHPPRO_T in this case, as the first member of the derived structure. See prepro member of RKHST_T structure for more information.

  • RKH_SMA_EN_HCAL

    If the RKH_SMA_EN_HCAL is set to 1, the RKH allows state nesting. When RKH_SMA_EN_HCAL is set to 0 some important features of RKH are not included: state nesting, composite state, history (shallow and deep) pseudostate, entry action, and exit action.

  • RKH_SMA_MAX_HCAL_DEPTH

    Specify the maximum number of hierarchical levels. The smaller this number, the lower the RAM consumption. Typically, the most of hierarchical state machines uses up to 4 levels. Currently RKH_SMA_MAX_HCAL_DEPTH cannot exceed 8.

  • RKH_SMA_MAX_TRC_SEGS

    Specify the maximum number of linked transition segments. The smaller this number, the lower the RAM consumption. Typically, the most of hierarchical state machines uses up to 4 transition segments. Currently RKH_SMA_MAX_TRC_SEGS cannot exceed 8.

  • RKH_SMA_EN_PSEUDOSTATE

    If the RKH_SMA_EN_PSEUDOSTATE is set to 1, the RKH allows pseudostates usage.

  • RKH_SMA_EN_DEEP_HISTORY

    If the RKH_SMA_EN_DEEP_HISTORY and RKH_SMA_EN_PSEUDOSTATE are set to 1, the RKH allows deep history pseudostate usage.

  • RKH_SMA_EN_SHALLOW_HISTORY

    If the RKH_SMA_EN_SHALLOW_HISTORY and RKH_SMA_EN_PSEUDOSTATE are set to 1, the RKH allows shallow history pseudostate usage.

  • RKH_SMA_EN_JUNCTION

    If the RKH_SMA_EN_JUNCTION and RKH_SMA_EN_PSEUDOSTATE are set to 1, the RKH allows junction pseudostate usage.

  • RKH_SMA_EN_CONDITIONAL

    If the RKH_SMA_EN_CONDITIONAL and RKH_SMA_EN_PSEUDOSTATE are set to 1, the RKH allows branch (or conditional) pseudostate usage.

  • RKH_SMA_EN_SUBMACHINE

    If the RKH_SMA_EN_SUBMACHINE and RKH_SMA_EN_PSEUDOSTATE are set to 1, the RKH allows submachine state (and exit/entry points) usage.

  • RKH_SMA_EN_INIT_ARG_SMA

    If the RKH_SMA_EN_INIT_ARG_SMA is set to 1 then the initial action prototype will add as argument a pointer to state machine structure RKHSMA_T. See RKHINIT_T definition.

  • RKH_SMA_EN_ENT_ARG_SMA

    If the RKH_SMA_EN_ENT_ARG_SMA is set to 1 then the entry action prototype will add as argument a pointer to state machine structure RKHSMA_T. See RKHENT_T definition.

  • RKH_SMA_EN_EXT_ARG_SMA

    If the RKH_SMA_EN_EXT_ARG_SMA is set to 1 then the exit action prototype will add as argument a pointer to state machine structure RKHSMA_T. See RKHEXT_T definition.

  • RKH_SMA_EN_ACT_ARG_SMA

    If the RKH_SMA_EN_ACT_ARG_SMA is set to 1 then the transition action prototype will add as argument a pointer to state machine structure RKHSMA_T. See RKHACT_T definition.

  • RKH_SMA_EN_ACT_ARG_EVT

    If the RKH_SMA_EN_ACT_ARG_EVT is set to 1 then the transition action prototype will add as argument a pointer to ocurred event. See RKHACT_T definition.

  • RKH_SMA_EN_GRD_ARG_EVT

    If the RKH_SMA_EN_GRD_ARG_EVT is set to 1 then the transition guard prototype will add as argument a pointer to ocurred event. See RKHGUARD_T definition.

  • RKH_SMA_EN_GRD_ARG_SMA

    If the RKH_SMA_EN_GRD_ARG_SMA is set to 1 then the transition guard prototype will add as argument a pointer to state machine structure RKHSMA_T. See RKHGUARD_T definition.

  • RKH_SMA_EN_PPRO_ARG_SMA

    If the RKH_SMA_EN_PPRO_ARG_SMA is set to 1 then the event preprocessor (Moore function like entry and exit actions) prototype will add as argument a pointer to state machine structure RKHSMA_T. See RKHPPRO_T definition.

Configuration options related to trace facility

  • RKH_TRC_EN

    If the RKH_TRC_EN is set to 1 then RKH will include the trace facility.

  • RKH_TRC_MAX_EVENTS

    Specify the maximum number of trace events, this number is direclty related with the RKH_TRC_EVENTS enumeration. The smaller this number, the lower the RAM consumption. See trceftbl table.

  • RKH_TRC_RUNTIME_FILTER

    If the RKH_TRC_RUNTIME_FILTER is set to 1 then RKH will include the runtime trace filter facility. When RKH_TRC_RUNTIME_FILTER is enabled RKH also will automatically define RKH_FILTER_ON_GROUP(), RKH_FILTER_OFF_GROUP(), RKH_FILTER_ON_EVENT(), RKH_FILTER_OFF_EVENT(), RKH_FILTER_ON_GROUP_EVENT(), and RKH_FILTER_OFF_GROUP_EVENT() macros.

  • RKH_TRC_ALL

    If the RKH_TRC_ALL is set to 1 then RKH will include all its own trace records.

  • RKH_TRC_EN_MP

    If the RKH_TRC_EN_MP is set to 1 then RKH will include all trace records related to the native fixed-size memory blocks.

  • RKH_TRC_EN_RQ

    If the RKH_TRC_EN_RQ is set to 1 then RKH will include all trace records related to the native queues.

  • RKH_TRC_EN_SMA

    If the RKH_TRC_EN_SMA is set to 1 then RKH will include all trace records related to the state machine applications.

  • RKH_TRC_EN_TIM

    If the RKH_TRC_EN_TIM is set to 1 then RKH will include all trace records related to the native software timer.

  • RKH_TRC_EN_SM

    If the RKH_TRC_EN_SM is set to 1 then RKH will include all trace records related to the state machine (hierarchical and "flat").

  • RKH_TRC_EN_RKH

    If the RKH_TRC_EN_RKH is set to 1 then RKH will include all trace records related to the nativenative event framework.

  • RKH_TRC_EN_SM_INIT

    If the RKH_TRC_EN_SM_INIT and RKH_TRC_EN_SM are set to 1 then RKH will include the "init state machine" trace record.

  • RKH_TRC_EN_SM_DCH

    If the RKH_TRC_EN_SM_DCH and RKH_TRC_EN_SM are set to 1 then RKH will include the "dispatch an event" trace record.

  • RKH_TRC_EN_SM_CLRH

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "clear the history pseudostate" trace record.

  • RKH_TRC_EN_SM_TRN

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "clear the history pseudostate" trace record.

  • RKH_TRC_EN_SM_STATE

    If the RKH_TRC_EN_SM_STATE and RKH_TRC_EN_SM are set to 1 then RKH will include the "final state of transition" trace record.

  • RKH_TRC_EN_SM_ENSTATE

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "entry state" trace record.

  • RKH_TRC_EN_SM_EXSTATE

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "exit state" trace record.

  • RKH_TRC_EN_SM_NENEX

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "number of entry and exit states in the transition" trace record.

  • RKH_TRC_EN_SM_NTRNACT

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "number of executed actions in the transition" trace record.

  • RKH_TRC_EN_SM_CSTATE

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "state or pseudostate in a compound transition" trace record.

  • RKH_TRC_EN_SM_DCH_RC

    If the RKH_TRC_EN_SM_CLRH and RKH_TRC_EN_SM are set to 1 then RKH will include the "returned code from dispatch function" trace record.

  • RKH_TRC_EN_NSEQ

    If the RKH_TRC_EN_NSEQ is set to 1 then RKH will add to the trace record an incremental number (1-byte), used like a sequence number. See RKH_TRC_NSEQ() and RKH_TRC_HDR() macros.

  • RKH_TRC_EN_CHK

    If the RKH_TRC_EN_CHK is set to 1 then RKH will add to the trace record a checksum (1-byte). See RKH_TRC_CHK() macro.

  • RKH_TRC_EN_TSTAMP

    If the RKH_TRC_EN_TSTAMP is set to 1 then RKH will add to the trace record a timestamp field. It's configurable by means of RKH_TRC_SIZEOF_TSTAMP.

  • RKH_TRC_SIZEOF_TSTAMP

    Specify the number of bytes (size) used by the trace record timestamp. The valid values [in bits] are 8, 16 or 32. Default is 16.

  • RKH_TRC_SIZEOF_STREAM

    Specify the maximum number of trace events in the stream. The smaller this number, the lower the RAM consumption.

  • RKH_TRC_SIZEOF_POINTER

    Specify the size of void pointer. The valid values [in bits] are 16 or 32. Default is 32. See RKH_TRC_SYM() macro.

Configuration options related to queue (by reference) facility

  • RKH_RQ_EN

    If the RKH_RQ_EN is set to 1 then RKH will include the native queue facility.

  • RKH_RQ_SIZEOF_NELEM

    Specify the maximum number of elements that any queue can contain. The valid values [in bits] are 8, 16 or 32. Default is 8. See RKH_RQNE_T type.

  • RKH_RQ_EN_GET_LWMARK

    If the RKH_RQ_EN_GET_LWMARK is set to 1 then RKH allows to known the minimum number of free elements ever in the queue (low-watermark ). This provides valuable empirical data for proper sizing of the queue. See rkh_rq_get_lwm() function.

  • RKH_RQ_EN_READ

    If the RKH_RQ_EN_READ is set to 1 then RKH will include the rkh_rq_read() function that allows read an element from a queue without remove it. See rkh_rq_read() function.

  • RKH_RQ_EN_DEPLETE

    If the RKH_RQ_EN_DEPLETE is set to 1 then RKH will include the rkh_rq_deplete() function that empties the contents of the queue and eliminates all stored elements. See rkh_rq_deplete() function.

  • RKH_RQ_EN_IS_FULL

    If the RKH_RQ_EN_IS_FULL is set to 1 then RKH will include the rkh_rq_is_full() function that allows to known if a queue is full. See rkh_rq_is_full() function.

  • RKH_RQ_EN_GET_NELEMS

    If the RKH_RQ_EN_GET_NELEMS is set to 1 then RKH will include the rkh_rq_get_num() function that returns the number of elements currently in the queue. See rkh_rq_get_num() function.

  • RKH_RQ_EN_PUT_LIFO

    If the RKH_RQ_EN_PUT_LIFO is set to 1 then RKH will include the rkh_rq_put_lifo() function that puts an element on a queue in a LIFO manner. See rkh_rq_put_lifo() function.

  • RKH_RQ_EN_GET_INFO

    If the RKH_RQ_EN_GET_INFO is set to 1 then RKH allows to collect and retrives performance information for a particular queue. See rkh_rq_get_info() and rkh_rq_clear_info() functions.

Configuration options related to fixed-sized memory block facility

  • RKH_MP_EN

    If the RKH_MP_EN is set to 1 then RKH will include the native fixed-size memory block management.

  • RKH_MP_REDUCED

    If the RKH_MP_EN is set to 1 then RKH will include the native fixed-size memory block management.

  • RKH_MP_SIZEOF_BSIZE

    Specify the size of memory block size. The valid values [in bits] are 8, 16 or 32. Default is 8. See RKH_MPBS_T type.

  • RKH_MP_SIZEOF_NBLOCK

    Specify size of number of memory block size. The valid values [in bits] are 8, 16 or 32. Default is 8. See RKH_MPNB_T type.

  • RKH_MP_EN_GET_BSIZE

    If the RKH_MP_EN_GET_BSIZE is set to 1 then RKH will include the rkh_mp_get_blksize() function that returns the size of memory block in bytes. See rkh_mp_get_blksize() function.

  • RKH_MP_EN_GET_NFREE

    If the RKH_MP_EN_GET_NFREE is set to 1 then RKH will include the rkh_mp_get_nfree() function that returns the current number of free memory blocks in the pool. See rkh_mp_get_nfree() function.

  • RKH_MP_EN_GET_LWM

    If the RKH_MP_EN_GET_LWM is set to 1 then RKH will include the rkh_mp_get_low_wmark() function that returns the lowest number of free blocks ever present in the pool. This number provides valuable empirical data for proper sizing of the memory pool. See rkh_mp_get_low_wmark() function.

  • RKH_MP_EN_GET_INFO

    If the RKH_MP_EN_GET_INFO is set to 1 then RKH will include the rkh_mp_get_info() function that retrieves performance information for a particular memory pool. See rkh_mp_get_info() function.

Configuration options related to software timer facility

  • RKH_TIM_EN

    If the RKH_RQ_EN is set to 1 then RKH will include the native software timer facility.

  • RKH_TIM_SIZEOF_NTIMER

    Specify the dynamic range of the time delays measured in clock ticks (maximum number of ticks). The valid values [in bits] are 8, 16 or 32. Default is 8.

  • RKH_TIM_EN_HOOK

    If the RKH_TIM_EN_HOOK is set to 1 then RKH will invoke a hook function when a timer expires. When this is set the application must provide the hook function.

  • RKH_TIM_EN_RESTART

    If the RKH_TIM_EN_RESTART is set to 1 then RKH will include the rkh_tim_restart() function that restarts a timer with a new number of ticks. See rkh_tim_restart() function.

  • RKH_TIM_EN_GET_INFO

    If the RKH_TIM_EN_GET_INFO is set to 1 then RKH will include the rkh_tim_get_info() function that retrieves performance information for a particular software timer. See rkh_tim_get_info() function.
